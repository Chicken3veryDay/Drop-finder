# Section-by-section recovery ledger

Create one report per completed section:

- `00_ARCHIVE_VERIFICATION.md`
- `00_CURRENT_BRANCH_DIFF.md`
- `01_PACKAGING.md`
- `02_CORE_SECURITY.md`
- `03_STORAGE_QUEUE.md`
- `04_FETCH_EVIDENCE.md`
- `05_PIPELINE.md`
- `06_RELIABILITY.md`
- `07_APP_INTERFACES.md`
- `08_TESTS_DOCS_LAUNCHERS.md`
- `09_PAGES_DEPLOYMENT.md`
- `10_FINAL_IMPORT_REPORT.md`

Each report must include files reviewed, findings, repairs, exact commands/tests, pass/fail counts, commit SHA, and known limitations.

The ledger prevents a bulk source dump, generated static output, or an experimental deployment workflow from being mistaken for a reviewed production repository.
